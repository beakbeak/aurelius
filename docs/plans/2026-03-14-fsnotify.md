# fsnotify Filesystem Watcher Plan

## Overview

After the initial full scan, the media database goes stale until restart. Add `fsnotify`-based filesystem watching so the DB stays in sync with the filesystem continuously.

Events are buffered with a configurable quiet period (default 5 seconds of no activity) before processing a batch. This ensures that move operations (which produce a Remove + Create pair) are captured together for proper move detection.

type WatcherConfig struct {
    // QuietPeriod is the duration of inactivity after the last event
    // before the batch is processed. Default: 5s.

type Watcher struct {

func NewWatcher(scanner *Scanner, rootPath string, config WatcherConfig) (*Watcher, error)

## Event Buffering and Debouncing

The watcher goroutine runs a select loop over `fsWatcher.Events`, `fsWatcher.Errors`, and a context done signal. A `time.Timer` implements the quiet period:

1. On each fsnotify event, record it in a `map[string]*pendingEvent` (keyed by absolute path) and reset the timer.
2. When the timer fires (no events for `QuietPeriod`), process the batch.
3. On context cancellation, stop immediately.

### Event Coalescing

Multiple events for the same path are coalesced within a batch:

```go
type eventKind int
const (
    eventCreated eventKind = iota
    eventModified
    eventRemoved
)
```

| Existing | New Event      | Result   | Rationale                              |
|----------|----------------|----------|----------------------------------------|
| (none)   | Create         | Created  |                                        |
| (none)   | Write          | Modified |                                        |
| (none)   | Remove/Rename  | Removed  |                                        |
| Created  | Write          | Created  | Still new, just being written          |
| Created  | Remove/Rename  | (delete) | Created then removed = no-op           |
| Modified | Write          | Modified |                                        |
| Modified | Remove/Rename  | Removed  |                                        |
| Removed  | Create         | Modified | Removed then re-created = changed      |

Chmod events are ignored entirely.

## Directory Handling

fsnotify does not watch recursively. We must manage subdirectory watches ourselves.

### Adding Initial Watches

After `FullScan()` completes, use `db.AllDirs()` to get all known directories and add a watch for each, plus the root. This avoids a redundant filesystem walk.

### New Directory Created

When a Create event fires and the path is a directory (checked via `os.Lstat`):

1. Skip if hidden (name starts with ".") or symlink.
2. Add an fsnotify watch on the directory.
3. Walk the directory recursively to discover files and subdirectories already present (handles `mv dir_from_outside/ watched_tree/` where only the top-level dir gets a Create event).
4. Add each discovered file/subdir to the pending batch as Created.
5. Add fsnotify watches for each discovered subdirectory.

### Directory Removed

fsnotify auto-removes watches on deleted directories. During batch processing, when a directory removal is detected, query `db.GetTracksInDir(dir)` and add any tracked files to `Removed` (handles atomic directory deletion where individual file Remove events may not fire).

### Identifying Directories in Remove Events

Since we cannot `stat` a removed path, maintain a set of watched directory paths. When a Remove/Rename event arrives, check this set to determine if the path was a directory.

## Building a ChangeSet from Buffered Events

When the quiet period expires, `processBatch` converts `map[string]*pendingEvent` into a `ChangeSet`:

1. **Convert paths**: `filepath.Rel(rootPath, absPath)` + `filepath.ToSlash` to get library-relative path. Split into `(dir, name)` using `path.Dir` / `path.Base` and `CleanLibraryPath`.

2. **Filter**: Skip hidden files/dirs. For files, skip if `GetFileType(name) != FileTypeTrack`. For Created/Modified, `os.Lstat` to confirm regular file (skip symlinks).

3. **Build ChangeSet**:
   - **Created files**: `os.Stat` for mtime, `computePartialHash` for hash. Add to `Added`.
   - **Modified files**: `os.Stat` for mtime. Add to `Changed`.
   - **Removed files**: `db.GetTrack(dir, name)` to get the full `Track` (with hash, needed for move detection). Add to `Removed`. Skip if not in DB.
   - **Created dirs**: Add to `AddedDirs`.
   - **Removed dirs**: Add to `RemovedDirs`. Query `db.GetTracksInDir` for orphaned tracks.

4. **Run existing pipeline**: `detectMoves(changes)` -> `scanner.collectMetadata(changes)` -> `scanner.Apply(result)`.

5. **Callback**: If `OnBatchApplied` is set, call it.

## Overflow Handling

When fsnotify reports a queue overflow, the OS dropped events and the watcher's view is inconsistent. Log a warning and trigger a full re-scan (`scanner.FullScan()`) to reconcile state.

## Integration with `pkg/media`

### Library Changes (`pkg/media/library.go`)

1. Add `watcher *mediadb.Watcher` field to `Library` struct.

2. In `NewLibrary`, after `FullScan()` and search index build are started, create and start the watcher:

```go
watcher, err := mediadb.NewWatcher(scanner, config.RootPath, mediadb.WatcherConfig{
    QuietPeriod: 5 * time.Second,
    OnBatchApplied: func() {
        go func() {
            if err := ml.searchIndex.BuildIndex(); err != nil {
                slog.Error("failed to rebuild search index", "error", err)
            }
        }()
    },
})
```

3. Add `Close()` method:

```go
func (ml *Library) Close() error {
    var firstErr error
    if ml.watcher != nil {
        if err := ml.watcher.Close(); err != nil && firstErr == nil {
            firstErr = err
        }
    }
    if err := ml.db.Close(); err != nil && firstErr == nil {
        firstErr = err
    }
    return firstErr
}
```

### Graceful Shutdown (`cmd/aurelius/main.go`)

Add signal handling so `Library.Close()` is called on SIGINT/SIGTERM:

```go
ml, err := media.NewLibrary(mlConfig)
if err != nil {
    log.Fatalf("failed to open media library: %v", err)
}
defer ml.Close()

srv := &http.Server{Addr: *listen, Handler: nil}

go func() {
    sigCh := make(chan os.Signal, 1)
    signal.Notify(sigCh, syscall.SIGINT, syscall.SIGTERM)
    <-sigCh
    slog.Info("shutting down")
    srv.Shutdown(context.Background())
}()
```
